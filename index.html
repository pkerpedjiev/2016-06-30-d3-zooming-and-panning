Example:

Let's plot 4 points: 
                    
    // create the svg
    var margin = {'left': 20, 'top': 20, 'bottom': 20, 'right': 20};
    var width = 400, height=60;
    var svg = d3.selectAll(".fig1")
        .attr('height', height + margin.top + margin.bottom)
        .attr('width', width + margin.left + margin.right);

    // create a container that will hold all of the graphics
    var gMain = svg.append('g')
        .attr('transform', 'translate(' + margin.left + ',' + margin.top + ')');

Example:

Let's plot 4 points: 
                    
    // create a scale for the data points
    var xScale = d3.scale.linear()
        .domain([0,5000])
        .range([0,width])

    // the actual data we want to plot
    var dataPoints = [1,1010,1020,5000];

    // create 'circle' svg elements for each point
    gMain.selectAll('circle')
    .data(dataPoints)
    .enter()
    .append('circle')
    .attr('r', 11)
    .attr('cx', function(d) { return xScale(d); });

Code:

                    
    var gZoom = gMain.append('g')

    var zoom = d3.behavior.zoom()
        .on('zoom', function() {
            gZoom.attr('transform', 'translate(' + zoom.translate()[0] + ',0)
                                     scale(' + zoom.scale() + ')')
        });
    gMain.call(zoom);

Geometric Zoom

Advantages:

  • Simple
  • Can increase the size of small details

Disadvantages

  • Leads to oversized glyphs
  • Doesn't resolve overlapping data

Can we do better?

  • Keep circles the same size
  • Increase the distance between them when zooming


This is semantic zooming

How did that work?

Circles are repositioned at each zoom event


    var zoom = d3.behavior.zoom()
        .x(xScale)
        .on('zoom', zoomed);

    function zoomed() {
        gAxis.call(xAxis);
        circles.attr('cx', function(d) { return xScale(d); });
    }

The zoom behavior changes the linear scale (xScale) and we use that to calculate new positions for each circle.

What?

Let's take a step back...


    var xScale = d3.scale.linear()
        .domain([0,5000])
        .range([0,400])

    ...

    gZoom.selectAll('circle')
    .data(dataPoints)
    .enter()
    .append('circle')
    .attr('r', 11)
    .attr('cx', function(d) { return xScale(d); });

We used a linear scale to convert the point's positions (from 0 to 5000) to positions in our viewport (from 0 to 400 pixels)

Linear Scales:


    var xScale = d3.scale.linear()
        .domain([0,5000])
        .range([100,500])

Map a domain interval to a range interval


So a point with value 0 has a position... 100

And a point with value 2500 has a position... 300

Zooming and Linear Scales

Zooming can change the domain of an associated scale:


    var zoom = d3.behavior.zoom()
        .x(xScale)
        .on('zoom', zoomed);

Another way of looking at it

Zooming increases the size of the viewport relative to the data
